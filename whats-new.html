v3.0.0 — Discover.

Five features that turn your library from a passive catalogue into an active research tool: search GitHub from inside the extension, get AI recommendations from your adopted repos, compare any N repos in a structured matrix, export a full decision matrix with eval scores and criteria, and a daily drift alert when repos go stale.

v3.0.0 ● NEWEST Discover. Jun 14
🔍

Discovery mode

Click "Discover" in the toolbar to open a GitHub search panel. Type any query — "rust async runtime", "react state management" — and get 12 starred results, filtered to repos you don't already have. One click to open and scan.

discoversearchgithub
✦

Recommendations

Click "Recommendations" in the discovery panel to get repos suggested from your own adopted/trial decisions. RepoLens extracts your top capabilities and primary language, then searches GitHub for similar repos you haven't seen yet.

recommendationsaidiscovery
⊞

N-way compare table

Select any 2–10 repos in selection mode and hit "⊞ Compare N". A full-screen matrix opens with every repo as a column and fit, health, stars, language, decision, eval score, per-rubric stars, capabilities, and note as rows. Export as Markdown or CSV directly from the modal.

comparematrixexport
📊

Decision matrix export

Two new palette commands export the full visible library as a structured matrix — CSV or Markdown — with every column: fit, health, stars, language, decision date, eval score, individual rubric criterion scores, eval note, and personal note. Ready for Notion, Obsidian, or a spreadsheet.

exportcsvmatrix
⏰

Drift alerts

A background alarm fires once a day, counts repos not scanned in 14+ days, and writes a summary. When you open the library a dismissable amber banner tells you exactly how many repos have gone stale — with a one-click "Refresh stale" shortcut.

driftalertsstale
🗂

Compare includes eval data

The N-way compare modal and both matrix exports include your rubric scores per criterion, so you can see not just "who scored higher" but "which repo is better documented" and "which has stronger type safety" side-by-side.

evalcomparerubric
