A good library shouldn't just remember what you've found — it should help you find what's next. **Discovery** turns RepoLens from a passive catalogue into an active research tool.

## Search GitHub from inside

Click **🔍 Discover** in the Library header and type a query — *"rust async runtime"*, *"react state management"*. RepoLens hits GitHub for the **top-starred matches** and shows the first dozen as cards (name, stars, language, blurb), automatically **hiding anything already in your library** so you only see new ground.

Found something promising? **Open & Scan ↗** opens its page so you can run a normal scan and fold it into your library — verdict, decision, and all.

## Recommendations from your own library

Don't have a query in mind? Hit **✦ Recommendations**. RepoLens looks at the repos you've already marked **Adopt** or **Trial**, pulls out your most common **capabilities** and your **primary language**, and searches GitHub for similar projects you haven't seen yet — *"recommended from your adopted/trial repos."*

The more you [triage](/docs/triage), the sharper the recommendations get: your decisions become the signal for what to surface next.

> Discovery uses GitHub's public search API and only reads repo metadata — no key required for this step, and nothing about your library leaves the browser. The optional AI step only runs when you actually scan a result.
